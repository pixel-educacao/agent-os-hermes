# Revisar saída

## Quando usar

Use para revisar material antes de entrega pública ou decisão importante.

## Passo a passo

1. Releia o objetivo original.
2. Avalie clareza, completude, consistência e risco.
3. Procure afirmações sem fonte.
4. Procure jargão, abstração e promessa exagerada.
5. Aponte cortes e melhorias.
6. Gere versão revisada se o usuário pediu execução.

## Checklist

- [ ] Está claro para o público-alvo?
- [ ] Tem exemplos concretos?
- [ ] Não promete o que não entrega?
- [ ] Não expõe dados privados?
- [ ] Tem próximo passo claro?
