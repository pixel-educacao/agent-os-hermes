# MEMORY.md — memória longa curada

Este arquivo guarda fatos estáveis que ajudam o agente a operar melhor.

Use com cuidado. Memória demais vira ruído.

## Princípios

- Salve fatos duráveis, não logs de sessão.
- Prefira resumo curado a transcrição bruta.
- Não salve segredos.
- Se for procedimento, vire skill.
- Se for tarefa, vá para `memory/tasks.md`.

## Memórias

- Este Agent OS usa root files como bootloader e skills como processos executáveis.
- O objetivo é transformar prompts soltos em sistemas de trabalho verificáveis.

## Lições recorrentes

- Prompt bom define objetivo, contexto, formato e critério de qualidade.
- Tarefa recorrente deve virar skill.
- Ação externa precisa de aprovação.
