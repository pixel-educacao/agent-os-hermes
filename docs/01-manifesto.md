# Manifesto — de Prompt Engineering para Work-System Design

Prompt engineering foi a primeira camada.

A próxima é desenhar sistemas de trabalho onde humanos e agentes colaboram com contexto, ferramentas, memória e critérios claros.

## A virada

O iniciante pergunta:

> Qual é o prompt certo?

O operador pergunta:

> Qual sistema faz este trabalho ficar repetível, verificável e melhor com o tempo?

## Prompt não morreu

Prompt continua importante. Mas prompt sozinho é frágil.

Um bom sistema inclui:

1. **Objetivo** — o que estamos tentando produzir.
2. **Contexto** — o que o agente precisa saber.
3. **Procedimento** — como executar.
4. **Ferramentas** — como acessar o mundo real.
5. **Memória** — o que deve sobreviver.
6. **Rails** — o que pode e não pode fazer.
7. **Verificação** — como saber se ficou certo.

## O papel do Agent OS

Agent OS é a camada entre uma pessoa e um agente.

Ele transforma:

```text
“faz isso pra mim”
```

em:

```text
leia o contexto certo → use o procedimento certo → execute → verifique → salve o que importa.
```

## O que muda na prática

Sem Agent OS:

- o usuário repete contexto;
- o agente esquece preferências;
- tarefas recorrentes recomeçam do zero;
- execução depende de sorte;
- ninguém sabe onde salvar nada.

Com Agent OS:

- root files dão direção;
- skills carregam processos;
- memória preserva continuidade;
- ferramentas verificam fatos;
- propagação evita perda de informação.

## Frase para lembrar

> Prompt bom é especificação.
> Agent OS é operação.
